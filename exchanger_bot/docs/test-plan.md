# Тест-план: Криптообменник (Telegram-боты)

**Проект:** Два Telegram-бота для обмена BTC ↔ RUB  
**Версия:** 1.0  
**Дата:** 2026-06-21  
**Автор:** Константин Горбунов (QA Engineer)

---

## 1. Цель тестирования

Проверить корректность работы клиентского и админского ботов, включая:
- Создание заявок на обмен
- Ввод реквизитов
- Отправку заявок оператору
- Ввод реквизитов оператором
- Отправку реквизитов клиенту
- Загрузку чеков
- Подтверждение оплаты
- Пагинацию по заявкам
- Кэширование курсов

---

## 2. Объекты тестирования

### Клиентский бот
| Модуль | Описание |
|:---|:---|
| **Главное меню** | 5 кнопок: "Создать обмен", "Курсы и лимиты", "Мои заявки", "Поддержка", "AML/KYC" |
| **Создание обмена** | Выбор валюты → ввод суммы → ввод реквизитов → подтверждение |
| **Курсы и лимиты** | Отображение актуальных курсов |
| **Мои заявки** | История заявок пользователя |
| **Загрузка чеков** | Отправка PDF, фото, документов |

### Админский бот
| Модуль | Описание |
|:---|:---|
| **Главное меню** | 4 кнопки: "Реквизиты на подтверждение", "Ожидают оплаты", "Оплаты на подтверждение", "Вкл/Выкл бот" |
| **Пагинация** | Стрелки влево/вправо по заявкам |
| **Ввод реквизитов** | Поэтапный ввод реквизитов оператора |
| **Подтверждение оплаты** | Подтверждение/отклонение оплаты |

---

## 3. Типы тестирования

| Тип | Описание |
|:---|:---|
| **Функциональное** | Проверка всех сценариев обмена |
| **UI/UX** | Проверка интерфейса ботов |
| **Интеграционное** | Проверка BestChange API, кэширования |
| **Регрессионное** | Проверка после изменений |

---

## 4. Инструменты тестирования

| Инструмент | Назначение |
|:---|:---|
| **Telegram** | Ручное тестирование ботов |
| **SQLite Browser** | Проверка данных в БД |
| **Postman** | Тестирование BestChange API |
| **Python** | Написание автотестов |

---

## 5. Критерии качества

- Все заявки создаются корректно
- Реквизиты сохраняются в БД
- Чеки пересылаются оператору
- Пагинация работает корректно
- Нет критических багов (P0)

---

## 6. Риски

| Риск | Митигация |
|:---|:---|
| Недоступность BestChange API | Кэширование курсов |
| Ошибки ввода данных | Валидация на стороне бота |
| Потеря данных | Резервное копирование БД |
