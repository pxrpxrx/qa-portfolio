# БАГ-РЕПОРТ #001

---

## 1. Заголовок
ConfigLoader падает с FileNotFoundError при отсутствии config.json

---

## 2. Метка приоритета
**Приоритет:** P1 (High)  
**Серьезность:** Critical

---

## 3. Описание
При запуске orchestrator.py или любого модуля, импортирующего config_loader, происходит падение с `FileNotFoundError`, если файл `config.json` отсутствует по пути `src/utilities/config/config.json`. Приложение не обрабатывает эту ситуацию и завершается с traceback.

---

## 4. Шаги для воспроизведения
1. Удалить или переименовать файл `src/utilities/config/config.json`
2. Запустить `python -m src.bx_orchestrator`
3. Наблюдать падение

---

## 5. Фактический результат
```
FileNotFoundError: [Errno 2] No such file or directory: '.../src/utilities/config/config.json'
```
Приложение аварийно завершается без внятного сообщения пользователю.

---

## 6. Ожидаемый результат
ConfigLoader должен обрабатывать отсутствие файла: логировать ошибку с понятным сообщением, использовать стандартные значения по умолчанию или предлагать создать шаблон конфига.

---

## 7. Окружение
- Python 3.10
- Windows 11
- Файл config.json не создан или удалён

---

## 8. Предполагаемая причина
В `config_loader.py:8` конструктор `ConfigLoader.__init__()` использует `open()` без блока `try/except FileNotFoundError`. При отсутствии файла исключение не перехватывается.

---

## 9. Ссылки
- Файл: `src/utilities/config_loader.py:8`
- Строка: `with open(config_path, 'r', encoding='utf-8') as f:`
