# Тест-план: Fractal Trader Bot

**Проект:** Автоматический торговый бот для BingX на основе фрактального анализа  
**Версия документа:** 1.0  
**Дата:** 2026-06-24  
**Автор:** Константин Горбунов (QA Engineer)

---

## 1. Введение

Цель тестирования — проверить корректность работы автоматического торгового бота Fractal Trader Bot. Система должна обеспечивать:

- Корректное обнаружение фракталов Уильямса (5-свечные паттерны)
- Точный расчёт размера позиции на основе ATR с риском 1% на сделку
- Надёжную работу риск-менеджера (дневные лимиты, чёрный список, просадка)
- Правильную запись и обновление сделок в SQLite
- Обработку ошибок и корректное поведение при сбоях API или конфигурации

---

## 2. Объекты тестирования

| Компонент | Описание |
|:---|:---|
| **positionSizing** | Расчёт размера позиции, стоп/тейк уровней, R/R на основе ATR |
| **RiskManager** | Глобальные лимиты, чёрный список, торговые часы, проверка R/R |
| **BingXDBWriter** | SQLite-логирование сделок (добавление, обновление, статистика) |
| **BingXOrderPreparer** | Парсинг сигналов и подготовка ордеров для BingX API |
| **FractalFinder** | Поиск фракталов на графике (верхних/нижних) |
| **ConfigLoader** | Загрузка конфигурации из JSON-файла |
| **BxMonitor** | Мониторинг открытых позиций, закрытие по стоп/тейк |
| **BingXAPI / BingXTrader** | Взаимодействие с BingX API |

---

## 3. Типы тестирования

| Тип | Описание |
|:---|:---|
| **Модульное тестирование** | Проверка каждого модуля изолированно (с mocking внешних зависимостей) |
| **Интеграционное тестирование** | Проверка связки modules (например, сигнал → парсер → ордер) |
| **Регрессионное тестирование** | Проверка после изменений (чек-лист прилагается) |
| **Тестирование безопасности** | Проверка обработки API-ключей, защита от утечек |
| **Тестирование обработки ошибок** | Поведение при отсутствии config.json, ошибках API |

## 5. Критерии качества

### Приемочные критерии (Definition of Done):
- ✅ Все заявленные функции работают корректно
- ✅ Нет критических багов (P0)
- ✅ Баг-репорты заведены для всех обнаруженных проблем
- ✅ Прохождение регрессионного чек-листа (100%)
- ✅ Покрытие тестами ключевых модулей > 80%

### Критерии остановки тестирования:
- Обнаружен критический баг, блокирующий работу основного функционала
- Недоступность BingX API > 30 минут
- Количество найденных P1-багов > 5

---

## 6. Риски и их митигация

| Риск | Митигация |
|:---|:---|
| Изменение BingX API | Мониторинг документации, версионирование эндпоинтов |
| Отсутствие config.json при запуске | Обработка FileNotFoundError с внятным сообщением |
| Некорректные сигналы от FRACTAL_SIGNALS | Валидация на уровне парсера, fallback-значения |
| Ошибки в расчёте размера позиции | Модульные тесты с фиксированными входными данными |
| Потеря соединения с БД | Обработка исключений, переподключение |
