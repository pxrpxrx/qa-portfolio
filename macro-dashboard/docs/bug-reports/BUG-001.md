# BUG-001: DataManager создаёт реальные HTTP-клиенты в конструкторе

## Описание
Конструктор `DataManager` создаёт реальные экземпляры `FredClient`, `CryptoClient`, `MacroDataClient`, что делает невозможным unit-тестирование без моков.

## Серьёзность
**Medium** — блокирует полное тестирование DataManager

## Шаги воспроизведения
1. Создать экземпляр `new DataManager()`
2. Проверить, что конструктор делает HTTP-запросы к внешним API

## Ожидаемое поведение
Конструктор должен принимать зависимости через параметры (Dependency Injection), что позволит подменять их моками в тестах.

## Фактическое поведение
Конструктор создаёт реальные клиенты:
```java
public DataManager() {
    this.fredClient = new FredClient();
    this.cryptoClient = new CryptoClient();
    this.macroDataClient = new MacroDataClient();
}
```

## Влияние
- Невозможно протестировать `DataManager.fetchAll()` без реальных API
- Тесты становятся медленными и нестабильными
- Нарушение принципа Dependency Injection

## Предлагаемое исправление
```java
public DataManager(FredClient fredClient, CryptoClient cryptoClient, MacroDataClient macroDataClient) {
    this.fredClient = fredClient;
    this.cryptoClient = cryptoClient;
    this.macroDataClient = macroDataClient;
}

// Для обратной совместимости
public DataManager() {
    this(new FredClient(), new CryptoClient(), new MacroDataClient());
}
```

## Статус
**Open** — требует рефакторинга

## Связанные компоненты
- `DataManager.java`
- `FredClient.java`
- `CryptoClient.java`
- `MacroDataClient.java`
