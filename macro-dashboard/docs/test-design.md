# Test Design — macro-dashboard

Применение Equivalence Partitioning и Boundary Value Analysis для макро-аналитического дашборда.

---

## 1. Equivalence Partitioning

### MacroContext (GREEN / YELLOW / RED)

Классы эквивалентности для агрегированной оценки:

| Partition | Условия | Представитель | Ожидаемый MacroContext |
|-----------|---------|---------------|------------------------|
| Все индикаторы позитивные | CPI ↓, безработица ↓, GDP ↑ | Snapshot с GREEN полями | GREEN |
| Смешанные сигналы | CPI ↑, безработица ↓, GDP → | Snapshot с mixed полями | YELLOW |
| Все индикаторы негативные | CPI ↑, безработица ↑, GDP ↓ | Snapshot с RED полями | RED |
| Пустые данные | Все null/NaN | Пустой Snapshot | YELLOW (дефолт) |

### DivergenceScanner

| Partition | Описание | Представитель | Ожидаемый результат |
|-----------|----------|---------------|---------------------|
| Сильная дивергенция | BTC растёт, альткоины падают | +3% BTC, -5% альты | HIGH divergence |
| Слабая дивергенция | Небольшое расхождение | +0.5% BTC, -0.3% альты | LOW divergence |
| Нет дивергенции | Всё движется вместе | +2% BTC, +2% альты | NO divergence |
| Нет данных | Пустые массивы | null prices | NO divergence (без crash) |

## Вопросы для собеседования

**Q: Как тестировать MacroContext (GREEN/YELLOW/RED)?**
A: Через EP — 3 класса (все позитивные, смешанные, все негативные) + edge case с пустыми данными. Для каждого класса — один Snapshot с нужными значениями.

**Q: Что такое дивергенция и как её тестировать?**
A: Дивергенция — расхождение между BTC и альткоинами. Тестируем через EP: сильная/слабая/нет/нет данных. BVA — на границе "есть/нет дивергенции" (расхождение ~0.1%).
