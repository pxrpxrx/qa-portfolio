# БАГ-РЕПОРТ #002

---

## 1. Заголовок
Расчет спреда показывает отрицательное значение при нормальных рыночных условиях

---

## 2. Метка приоритета
**Приоритет:** P1 (High)  
**Серьезность:** Major

---

## 3. Описание
При мониторинге спреда между биржами MEXC и Bybit бот иногда выдает отрицательное значение спреда (например, -2.34%), хотя разница цен объективно положительная.

---

## 4. Шаги для воспроизведения
1. Запустить воркфлоу Spread Count
2. Дождаться расчета спреда для пары MEXC-BYBIT
3. Проверить значение в JSON-ответе

---

## 5. Фактический результат
Значение спреда = -2.34% (отрицательное)

---

## 6. Ожидаемый результат
Значение спреда всегда >= 0 (положительное или 0)

---

## 7. Окружение
- n8n 0.238.0
- JavaScript (Code-нода SPREAD COUNT8)
- API: MEXC, Bybit

---

## 8. Предполагаемая причина
Ошибка в логике сравнения: перепутаны местами bidPrice и askPrice. 
В коде: `bestBuy` берет максимальную цену, `bestSell` — минимальную.

---

## 9. Код для проверки
```javascript
// Некорректная логика
if (ex.buyPrice > bestBuy.price) { // ДОЛЖНО БЫТЬ <
    bestBuy.price = ex.buyPrice;
}
