# Тест-план: Crypto Arbitrage Bot

**Проект:** Telegram-бот для мониторинга межбиржевых спредов криптовалют  
**Версия документа:** 1.0  
**Дата:** 2025-06-21  
**Автор:** Константин Горбунов (QA Engineer)

---

## 1. Введение

Цель тестирования — проверить корректность работы Telegram-бота для мониторинга межбиржевых спредов криптовалют. Система должна обеспечивать:

- Корректный сбор данных с API бирж (BingX, MEXC, Bybit, HTX, Gate, Bitget, OKX)
- Точный расчет спреда в реальном времени
- Правильную работу Telegram-интерфейса (кнопки, команды, уведомления)
- Надежное хранение пользовательских настроек в PostgreSQL
- Обработку ошибок и корректное поведение при сбоях

---

## 2. Объекты тестирования

| Компонент | Описание |
|:---|:---|
| **Telegram Front** | Интерфейс взаимодействия с пользователем (кнопки, команды, ответы) |
| **Spread Count** | Логика сбора данных с API бирж и расчета спреда |
| **User Settings** | Механизм сохранения и обновления настроек пользователя |
| **Network Updater** | Ежесуточное обновление данных о сетях и комиссиях |
| **Database** | PostgreSQL — хранение настроек, данных о сетях, состояния пользователей |
| **Payment Redirect** | Интеграция с платежным модулем (редирект на сайт оплаты) |

---

## 3. Типы тестирования

| Тип | Описание |
|:---|:---|
| **Функциональное тестирование** | Проверка всех заявленных функций бота |
| **Интеграционное тестирование** | Проверка взаимодействия с API бирж и базой данных |
| **Регрессионное тестирование** | Проверка после изменений (чек-лист прилагается) |
| **UI/UX тестирование** | Проверка Telegram-интерфейса (кнопки, сообщения, навигация) |
| **Тестирование безопасности** | Проверка защиты API-ключей, приватных данных |
| **Тестирование производительности** | Проверка времени ответа API и расчета спреда |

## 4. Инструменты тестирования

| Инструмент | Назначение |
|:---|:---|
| **Postman** | Ручное и автоматизированное API-тестирование (биржи, Telegram) |
| **DBeaver** | Проверка данных в PostgreSQL (настройки, логи) |

---

## 5. Критерии качества

### Приемочные критерии (Definition of Done):
- ✅ Все заявленные функции работают корректно
- ✅ Нет критических багов (P0)
- ✅ Баг-репорты заведены для всех обнаруженных проблем
- ✅ Прохождение регрессионного чек-листа (100%)
- ✅ API бирж отвечают с таймаутом не более 5 секунд

### Критерии остановки тестирования:
- Обнаружен критический баг, блокирующий работу основного функционала
- Недоступность одной из бирж > 30 минут
- Количество найденных P1-багов > 10
