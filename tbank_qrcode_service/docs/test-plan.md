# План тестирования микросервиса TBank QR Code Service

## 1. Объект тестирования

Микросервис **TBank QR Code Service** — приём платежей через СБП Тинькофф с генерацией QR-кодов.

**Адрес:** http://208.92.226.223:8080  
**WebSocket:** ws://208.92.226.223:8080/ws

## 2. Цели тестирования

- Проверить корректность генерации QR-кода и инициализации платежа
- Проверить валидацию входных параметров
- Проверить WebSocket-уведомления об оплате
- Проверить корректную обработку ошибок и граничных случаев
- Проверить стабильность при нагрузке

## 3. Объекты тестирования

| Объект | Метод | Описание |
|--------|-------|----------|
| REST API | `GET /api/sbp/init-and-qr` | Инициализация платежа, получение QR |
| WebSocket | `/ws` → `/topic/payment/{orderId}` | Уведомления о статусе |
| Actuator | `GET /actuator/health` | Health check |

## 4. Типы тестов

| Тип | Инструмент | Описание |
|-----|-----------|----------|
| Модульные (unit) | JUnit 5 + Mockito | Тестирование сервисов изолированно |
| Интеграционные (Spring) | Spring Boot Test + MockMvc | Тестирование контроллеров |
| API-тесты | JUnit 5 + HttpClient | Черный ящик, реальный сервер |
| WebSocket-тесты | JUnit 5 + Spring WebSocketClient | Подписка, получение уведомлений |
| Ручные | Postman / curl | Исследовательское тестирование |

## 5. Инструменты

- **Java:** JUnit 5, Mockito, Spring Boot Test, MockMvc, HttpClient, Spring WebSocketClient
- **Постман:** Postman collection для ручного тестирования
- **Maven:** `mvn test` для Java-тестов
- **Docker Compose:** PostgreSQL + Redis для интеграционных тестов
- **k6:** Нагрузочное тестирование (`performance-tests/load-test-tbank.js`)

## 6. Критерии качества

- **P0 (Критические):** 100% прохождение — инициализация платежа с корректными параметрами
- **P1 (Высокие):** 100% прохождение — валидация обязательных параметров
- **P2 (Средние):** минимальные отклонения допустимы — граничные значения, спецсимволы

## 7. Тестовые сценарии

| ID | Название | Приоритет |
|----|----------|-----------|
| TC-001 | Инициализация платежа с корректными параметрами | P0 |
| TC-002 | Инициализация платежа без userId | P1 |
| TC-003 | Инициализация платежа с отрицательной суммой | P2 |
| TC-004 | Подключение к WebSocket и получение статуса оплаты | P0 |

## 8. Покрытие тестами

### Unit-тесты (15 тестов)
- **TinkoffApiServiceTest** (5 тестов): валидация параметров, создание статуса в Redis
- **PaymentStatusStorageTest** (7 тестов): CRUD операции с Redis, TTL, финальные статусы
- **QrCodeServiceTest** (5 тестов): генерация QR-кодов, валидация входных данных
- **SbpControllerTest** (4 теста): обработка запросов, ошибки Tinkoff API

### Интеграционные тесты
- **PaymentApiIntegrationTest** (15 тестов): REST API (требует запущенный сервер)
- **WebSocketIntegrationTest** (4 теста): WebSocket подписки (требует запущенный сервер)

### Нагрузочные тесты
- **k6**: `performance-tests/load-test-tbank.js` (требует запущенный сервер)

## 8. Окружение

- **Стенд:** http://208.92.226.223:8080
- **БД:** PostgreSQL 14, `payments_db`
- **Кэш:** Redis 6
- **Внешние зависимости:** Tinkoff API (https://securepay.tinkoff.ru/v2/)
