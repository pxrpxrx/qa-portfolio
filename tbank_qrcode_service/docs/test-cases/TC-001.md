# TC-001: Инициализация платежа с корректными параметрами

**Приоритет:** P0 (Критический)  
**Тип:** Позитивный / Smoke  
**Метод:** GET  
**Endpoint:** `/api/sbp/init-and-qr`

## Предусловия

- Микросервис запущен и доступен по адресу http://208.92.226.223:8080
- Tinkoff API доступен

## Шаги

1. Отправить GET-запрос:
   ```
   GET /api/sbp/init-and-qr?userId=test_user_001&amount=100.50
   ```

## Ожидаемый результат

- **Статус:** 200 OK
- **Тело ответа (JSON):**
  ```json
  {
    "qrSvg": "<svg>...</svg>",
    "orderId": "order-xxxxxxxx-xxxx"
  }
  ```
- `orderId` — непустая строка
- `qrSvg` — непустая строка, начинается с `<svg` или `data:image`

## Постусловия

- Создан заказ в БД (PaymentEntity)
- Статус заказа сохранён в Redis
