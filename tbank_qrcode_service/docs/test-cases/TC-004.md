# TC-004: Подключение к WebSocket и получение статуса оплаты

**Приоритет:** P0 (Критический)  
**Тип:** Smoke / WebSocket  
**Метод:** WebSocket  
**Endpoint:** `ws://208.92.226.223:8080/ws`

## Предусловия

- Микросервис запущен и доступен
- Получен orderId через REST (TC-001)

## Шаги

1. Получить orderId через `GET /api/sbp/init-and-qr`
2. Подключиться к WebSocket: `ws://208.92.226.223:8080/ws`
3. Отправить STOMP-подписку:
   ```
   SUBSCRIBE
   id:sub-0
   destination:/topic/payment/{orderId}
   ```
4. Оплатить заказ через Tinkoff (внешний шаг)
5. Дождаться сообщения от WebSocket

## Ожидаемый результат

- WebSocket-соединение установлено (статус `CONNECTED`)
- Подписка подтверждена
- При оплате приходит JSON:
  ```json
  {
    "orderId": "order-xxxxxxxx-xxxx",
    "status": "CONFIRMED"
  }
  ```
- `status` имеет значение `CONFIRMED`

## Постусловия

- WebSocket-соединение закрыто
