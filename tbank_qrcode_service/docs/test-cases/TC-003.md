# TC-003: Инициализация платежа с отрицательной суммой

**Приоритет:** P2 (Средний)  
**Тип:** Негативный / Валидация  
**Метод:** GET  
**Endpoint:** `/api/sbp/init-and-qr`

## Предусловия

- Микросервис запущен и доступен

## Шаги

1. Отправить GET-запрос с отрицательной суммой:
   ```
   GET /api/sbp/init-and-qr?userId=test_user_003&amount=-50
   ```

## Ожидаемый результат

- **Статус:** 400 Bad Request
- Сервер отклоняет запрос с отрицательной суммой

## Постусловия

- Платеж в Tinkoff не создан

## Связанные дефекты

- BUG-001: Отсутствует валидация amount на отрицательные значения
