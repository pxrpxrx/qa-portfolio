# BUG-001: Отсутствует валидация amount на отрицательные значения

**Приоритет:** P1 (Major)  
**Статус:** Открыт  
**Компонент:** REST API / Контроллер  
**Endpoint:** `GET /api/sbp/init-and-qr`

## Описание

При передаче отрицательного значения в параметре `amount` сервер возвращает 200 OK и
пытается инициализировать платеж в Tinkoff API, который в свою очередь возвращает ошибку.
Правильным поведением было бы возвращать 400 Bad Request на уровне приложения.

## Шаги воспроизведения

1. Отправить запрос:
   ```
   GET /api/sbp/init-and-qr?userId=test_user&amount=-100
   ```

## Фактический результат

- Статус: 200 OK (или 500, если Tinkoff вернул ошибку)
- Платеж не создан, но нет понятной ошибки валидации

## Ожидаемый результат

- Статус: 400 Bad Request
- Сообщение: "Amount must be positive"

## Окружение

- Стенд: http://208.92.226.223:8080
- Версия: payment-api-3.0.jar

## Возможное решение

Добавить `@Min(1)` или кастомный валидатор на параметр `amount` в контроллере:

```java
@RequestParam @Min(value = 1, message = "Amount must be positive") double amount
```
