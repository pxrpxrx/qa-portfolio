# BUG-002: Чувствительные данные в application.properties внутри JAR

| Поле | Значение |
|------|----------|
| **ID** | BUG-002 |
| **Проект** | tbank_qrcode_service |
| **Приоритет** | P1 (High) |
| **Серьёзность** | Critical |
| **Статус** | Open |
| **Автор** | QA Engineer |
| **Дата** | 2026-06-24 |

## Описание

В скомпилированном JAR-файле (`src/payment-api-3.0.jar`) содержатся незашифрованные учётные данные:

- `tinkoff.secret.key=KPFvJO5lRMypTV*y` — секретный ключ API Тинькофф
- `spring.datasource.password=12345678` — пароль к PostgreSQL

Это означает, что любой, у кого есть доступ к JAR, может получить эти данные.

## Шаги воспроизведения

1. Скачать JAR: `src/payment-api-3.0.jar`
2. Распаковать: `unzip -l payment-api-3.0.jar`
3. Найти `application.properties`
4. Прочитать учётные данные в открытом виде

## Ожидаемый результат

Учётные данные не должны храниться в репозитории. Должен использоваться внешний файл конфигурации или переменные окружения.

## Фактический результат

Секретный ключ Тинькофф и пароль БД зашиты в JAR в незашифрованном виде.

## Рекомендация

1. Удалить JAR из репозитория (добавить в .gitignore)
2. Использовать `application.properties.template` в репозитории
3. Передавать реальные secrets через переменные окружения или внешний config-файл
4. Пересобрать JAR без встроенных секретов
