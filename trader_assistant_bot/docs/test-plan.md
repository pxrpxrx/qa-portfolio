# Тест-план: ATR Trailing Stop Bot (BingX)

**Проект:** Автоматизированный торговый бот для BingX с ATR-трейлинг стопами  
**Версия документа:** 1.0  
**Дата:** 2025-06-24  
**Автор:** QA Engineer

---

## 1. Введение

Цель тестирования — проверка корректности работы автоматизированного торгового бота для BingX. Система должна обеспечивать:

- Точный расчёт ATR (Average True Range) на исторических данных
- Корректное открытие/закрытие позиций со стоп-лоссом и тейк-профитом
- Мониторинг позиций с принятием решений по PnL (Stop Loss / Take Profit)
- Соблюдение риск-лимитов: дневной убыток, чёрный список, торговые часы
- Обработку ошибок API и краевых случаев (недостаточно данных, нулевой баланс)

## 2. Объекты тестирования

| Компонент | Описание |
|:---|:---|
| **ATR (bx_atr.py)** | Расчёт ATR, ATR×3, ATR%, позиционирование |
| **BingXTrader (bx_trader.py)** | Полный цикл ордеров: открытие, стопы, тейки, закрытие, обновление |
| **BxMonitor (bx_monitor.py)** | Мониторинг позиций, принятие решений Stop Loss / Take Profit |
| **RiskManager (bx_risk_manager.py)** | Глобальные риск-лимиты, чёрный список, торговые часы |
| **BingXKlineData (klineData.py)** | Получение и парсинг свечных данных с BingX |
| **BingXAPI (bx_api.py)** | HMAC-подписанный HTTP-клиент для BingX |
| **ConfigLoader (config_loader.py)** | Загрузка конфигурации из JSON |

## 3. Типы тестирования

| Тип | Описание |
|:---|:---|
| **Модульное тестирование** | Проверка каждого компонента изолированно (с моками зависимостей) |
| **Интеграционное тестирование** | Взаимодействие модулей (Trader → API, Monitor → Trader) |
| **Регрессионное тестирование** | Проверка после изменений (чек-лист прилагается) |
| **Тестирование граничных случаев** | Пустые данные, ошибки API, лимиты, границы диапазонов |
| **Тестирование безопасности** | Проверка экранирования API-ключей, отсутствия утечек секретов |

## 5. Критерии качества

### Приемочные критерии (Definition of Done):
- ✅ Все P0 тест-кейсы пройдены
- ✅ Покрытие кода тестами не менее 70%
- ✅ Нет критических багов (P0)
- ✅ Баг-репорты заведены для всех обнаруженных проблем
- ✅ Прохождение регрессионного чек-листа (100%)

### Критерии остановки тестирования:
- Обнаружен критический баг, блокирующий тестирование (например, падение ConfigLoader)
- Отсутствие зависимостей (bx_fractal.py, positionSizing.py)
- Количество P1-багов > 5

## 7. Риски и их митигация

| Риск | Митигация |
|:---|:---|
| Отсутствие bx_fractal.py, positionSizing.py в репозитории | Мокирование внешних зависимостей в тестах |
| Изменение API BingX (эндпоинты, формат ответа) | Использование моков, периодическая валидация через Postman |
| ConfigLoader падает без config.json | Тест BUG-001, graceful fallback |
| Rate-лимиты BingX при интеграционных тестах | Тесты на rate limiting в monitor |
| Некорректный расчёт ATR на реальных данных | Множественные тесты с разными наборами mock-данных |
